** Introduction

In most presentations of Riemannian geometry, e.g. [cite/t:@o1983semi] and [[https://en.wikipedia.org/wiki/Fundamental_theorem_of_Riemannian_geometry][Wikipedia]],
the fundamental theorem of Riemannian geometry ("the miracle of
Riemannian geometry") is given: that for any semi-Riemannian manifold
there is a unique torsion-free metric connection. I assume partly
because of this and partly because the major application of Riemannian
geometry is General Relativity, connections with torsion are given
little if any attention.

It turns out we are all very familiar with a connection with torsion:
the Mercator projection. Some mathematical physics texts,
e.g. [cite/t:@Nakahara2003], allude to this but leave the details to the
reader. Moreover, this connection respects the metric induced from
Euclidean space.

The connection we construct cannot exist on the entire sphere as we
are going to need to define a orthogonal frame and by the [[https://en.wikipedia.org/wiki/Hairy_ball_theorem][hairy ball
theorem]] such a frame cannot exist on all of \(S^2\). We will instead
define the connection on an open subset and for convenience let us
take that to be the spherical coordinate chart on \(S^2\), that is the map
\((\theta, \phi) \mapsto (\sin\theta \cos\phi, \sin\theta \sin\phi, \cos\theta)\),
with \(\theta \in (0, \pi)\), the angle from the north pole and \(\phi \in (0, 2\pi)\) the
longitude. In Lean we can write the inverse chart map as

#+begin_src lean4 :tangle no
def sphSource : Set S2 := {p | 0 < p.val 0 ∨ p.val 1 ≠ 0}

noncomputable def sphInvVec (q : EuclideanSpace ℝ (Fin 2)) : EuclideanSpace ℝ (Fin 3) :=
    (WithLp.equiv 2 (Fin 3 → ℝ)).symm
      ![Real.sin (q 0) * Real.cos (q 1),
	Real.sin (q 0) * Real.sin (q 1),
	Real.cos (q 0)]
#+end_src

NB this is not the Mercator projection; we will soon define that.

First suppose we have the round metric on the sphere:

\[
g = d\theta^2 + \sin^2\theta \, d\phi^2,
\qquad (\theta, \phi) \in (0, \pi) \times (0, 2\pi)
\]

We know if we use the Levi-Civita connection then there are no
parallel lines as great circles will always intersect at two points.
Instead let us change what parallel means: two tangent vectors are
parallel if they have the same compass bearing, that is, they make the same
angle with a line of longitude.

** The Compass Frame

For a bearing to make sense we must have some concept of direction. Recall
the metric at a point is an inner product \(g_x\) on the tangent space
\(T_xS^2\), and the angle between nonzero \(v, w\) is defined by

\[
\cos\angle_x(v,w)=
\frac{g_x(v,w)}
{\sqrt{g_x(v,v)}\sqrt{g_x(w,w)}}.
\]

A bearing south is \(\partial\theta\) (since since θ is the angle from
the north pole) and the angle of a bearing to a meridian is

\[
\cos\angle_x(\partial\theta,v)
\]

so we should take that as one component of our frame \(e_1 =
\partial\theta\) and then we are forced by the metric to choose \(e_2
= (1 / \sin{\theta})\partial\phi\).

Now we can define two vectors to be parallel if their angles to a

** Connections from Coefficients in a Frame

In SageManifolds we can just say =nabla.add_coef(e)= to specify a
covariant derivative by its connection coefficients =Γ= relative to a
frame =e= with dual coframe =ε=, supported on a set =s=. Well the
coframe and the valid set do not need to specified (why?).

Let \(\nabla\) be any Koszul connection thus Leibniz in the second argument:

\[\nabla_v(f\sigma) = (df|_x(v))\,\sigma(x) + f(x)\,\nabla_v\sigma\]

Let \(e_i\) be a frame with corresponing coframe \(\varepsilon^i\) on some open set \(S\), and
define the connexion coefficients by

\[
\nabla_{e_j} e_k = \Gamma^i_{jk}\, e_i
\]

Now take any section \(\sigma\) (think vector field) and expand it in the frame:

\[
\sigma = \sum_k \varepsilon^k(\sigma)\, e_k
\]

For \(v \in T_xM\), additivity and then the Leibniz
rule give

\[
\nabla_v \sigma =
\sum_k  \nabla_v\big(\varepsilon^k(\sigma)\, e_k\big) =
\sum_k d(\varepsilon^k(\sigma))\big|_x(v)\, e_k(x) +
\varepsilon^k(\sigma)(x)\, \nabla_v e_k
\]

Writing
\(v = \sum_j \varepsilon^j_x(v)\, e_j(x)\):

\[
\nabla_v e_k
= \sum_j \varepsilon^j_x(v)\, \nabla_{e_j} e_k
= \sum_{i,j} \varepsilon^j_x(v)\, \Gamma^i_{jk}(x)\, e_i(x).
\]

Substituting back and relabelling:

\[
(\nabla\sigma)(x)(v) =
\sum_i \Big( d(\varepsilon^i(\sigma))\big|_x(v) +
\sum_{j,k} \Gamma^i_{jk}(x)\, \varepsilon^k(\sigma)(x)\, \varepsilon^j_x(v) \Big)\, e_i(x)
\]

Conversely, any smooth choice of \(\Gamma^i_{jk}\) makes this formula a
Koszul connection: additivity is immediate and the Leibniz rule is a
one-line check; so on \(S\), connexions correspond exactly to the connexion
coefficients relative to the frame.

We can write this definition for the specific case of \(S^2\) in Lean as:

#+begin_src lean4 :tangle MercatorConnection/Posts/MercatorVPostCode.lean

  def covDerivOfFrame (s : Set S2)
      (e : Fin 2 → Π x : S2, TangentSpace (𝓡 2) x)
      (ε : Fin 2 → Π x : S2, TangentSpace (𝓡 2) x →L[ℝ] ℝ)

#+begin_src lean4 :tangle MercatorConnection/Posts/MercatorVPostCode.lean
lemma frame_pairing_pre {x : S2} (hx : x ∈ sphSource) :
    (mfderiv (𝓡 2) 𝓘(ℝ, EuclideanSpace ℝ (Fin 2)) sphFwd x) ∘SL (mfderiv 𝓘(ℝ, EuclideanSpace ℝ (Fin 2)) (𝓡 2) sphInv (sphFwd x)) =
     ContinuousLinearMap.id ℝ (EuclideanSpace ℝ (Fin 2)) := by
  set Dfwd : TangentSpace (𝓡 2) x →L[ℝ] EuclideanSpace ℝ (Fin 2) :=
    mfderiv (𝓡 2) 𝓘(ℝ, EuclideanSpace ℝ (Fin 2)) sphFwd x
  set Dinv : EuclideanSpace ℝ (Fin 2) →L[ℝ] TangentSpace (𝓡 2) x :=
    mfderiv 𝓘(ℝ, EuclideanSpace ℝ (Fin 2)) (𝓡 2) sphInv (sphFwd x)
  -- right inverse: `Dfwd ∘ Dinv = id`, mirroring `h_Dinv_Dfwd` in `frame_dual`
  have hxx : sphInv (sphFwd x) = x := sph_left_inv x hx
  have h_right : Dfwd.comp Dinv = ContinuousLinearMap.id ℝ (EuclideanSpace ℝ (Fin 2)) := by
    have h_inv : ∀ᶠ q in nhds (sphFwd x), sphFwd (sphInv q) = q :=
      Filter.eventually_of_mem (IsOpen.mem_nhds sph_open_target (sph_map_source hx))
        fun q hq => sph_right_inv q hq
    have hid : mfderiv 𝓘(ℝ, EuclideanSpace ℝ (Fin 2)) 𝓘(ℝ, EuclideanSpace ℝ (Fin 2))
        (sphFwd ∘ sphInv) (sphFwd x)
        = ContinuousLinearMap.id ℝ (EuclideanSpace ℝ (Fin 2)) :=
      HasMFDerivAt.mfderiv
        (HasMFDerivAt.congr_of_eventuallyEq (hasMFDerivAt_id _) h_inv)
    have hg : MDifferentiableAt (𝓡 2) 𝓘(ℝ, EuclideanSpace ℝ (Fin 2)) sphFwd
        (sphInv (sphFwd x)) := by
      rw [hxx]; exact sphFwd_mdiffAt hx
    have hf : MDifferentiableAt 𝓘(ℝ, EuclideanSpace ℝ (Fin 2)) (𝓡 2) sphInv (sphFwd x) :=
      sphInv_contMDiff.contMDiffAt.mdifferentiableAt (by norm_num)
    have hcomp := mfderiv_comp (sphFwd x) hg hf
    rw [hxx] at hcomp
    exact hcomp.symm.trans hid
  exact h_right

** Covariant Derivative

\[
\text{For } x \in U \text{ and } v \in T_xS^2:\qquad
v = \varepsilon^1(v)\, e_1 + \varepsilon^2(v)\, e_2
\]

\[
v = \varepsilon^1_x(v)\, e_1(x) + \varepsilon^2_x(v)\, e_2(x)
\qquad \text{for all } x \in U,\ v \in T_x S^2
\]

\[
v =
d\theta(v)\,\partial_\theta + \big(\sin\theta\, d\varphi(v)\big)\,\frac{1}{\sin\theta}\,\partial_\varphi
\]
